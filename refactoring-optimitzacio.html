1. Què és el Refactoring?

El refactoring és el procés de millorar el codi internament sense canviar el seu comportament extern.

🎯 Objectiu:

  • >> Fer-lo més llegible
  • >> Fer-lo més mantenible
  • >> Reduir duplicacions
  • >> Preparar-lo per créixer

📌 Important:

El refactoring NO afegeix funcionalitats noves.

Exemple simple:

Millorable:

int vidaJugador = 100;
if (vidaJugador <= 0) {
    Console.WriteLine("Game Over");
}

Refactoritzat:

private static void ComprovarMort(int vida) {
    if (vida <= 0) {
        Console.WriteLine("Game Over");
    }
}

2. Què és l’Optimització?

L’optimització busca millorar el rendiment del programa per estalviar recursos del sistema.

  • >> Reduir ús de CPU
  • >> Reduir ús de memòria
  • >> Millorar FPS en videojocs
  • >> Evitar càlculs innecessaris

📌 Diferència clau:

3. Refactoring: Casos típics

❌ Codi duplicat:

Repetir el càlcul del dany en diversos llocs.

public static int CalcularDany(int atac, int defensa) {
    int dany = atac - defensa;
    return dany < 0 ? 0 : dany;
}

❌ Mètodes massa llargs:

Evita mètodes de 150 línies. Divideix en funcions modulars.

❌ Noms pobres:

Canvia int x, y, z; per noms descriptius.

4. Eines de Refactoring en VS

  • >> Rename: Canviar nom de forma segura.
  • >> Extract Method: Convertir codi en mètode.
  • >> Inline variable: Eliminar variables redundants.

Quick Actions:

Usa la bombeta (Ctrl + .) per veure suggeriments automàtics de l'IDE.

5. Quan refactoritzar?

✔ Quan el codi funciona.

✔ Quan detectem duplicacions.

✔ Quan el codi costa d’entendre.
