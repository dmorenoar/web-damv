🧠 Nivell 5 — Modularitat i Mètodes

Divideix i venceràs: arquitectura de codi professional

1. Què és la modularitat?

La modularitat és el principi de dividir un programa gran en parts més petites i independents anomenades mòduls o mètodes.

🎯 Objectiu:

  • >> Fer el codi més llegible
  • >> Facilitar el manteniment
  • >> Reutilitzar funcionalitats
  • >> Millorar el debugging

EN VIDEOJOCS:

Cada part ha d'estar separada: Sistema de vida, Moviment, Combat, IA d'enemics, Puntuació.

2. Què és un mètode?

Un mètode és un bloc de codi que realitza una tasca concreta. En C#, un mètode té aquesta estructura:

ModificadorAcces TipusRetorn NomMetode(Paràmetres)
{
    // Cos del mètode
}

Exemple:

public void MostrarMissatge()
{
    Console.WriteLine("Has iniciat el joc!");
}

3. Parts d'un mètode

🔐 Modificador d’accés: public, private, protected.

🔁 Tipus de retorn: void (no retorna res), int, float, bool, string...

🏷 Nom del mètode: Segueix PascalCase (ex: CalcularDany()).

📥 Paràmetres: Permeten enviar informació al mètode.

4. Tipus de mètodes

🔹 Mètodes sense retorn (void)

Fan una acció però no retornen cap valor (sons, animacions, UI).

public void ReproduirSoImpacte()
{
    Console.WriteLine("So d'impacte!");
}

🔹 Mètodes amb retorn

Retornen un valor calculat (lògica matemàtica de combat).

public int CalcularDany(int atac, int defensa)
{
    int danyFinal = atac - defensa;
    return danyFinal;
}

5. Codi sense modularitzar

Imaginem un sistema d’atac escrit tot dins del Main:

int atac = 50, defensa = 20, vidaEnemic = 100;
int dany = atac - defensa;
if (dany < 0) dany = 0;
vidaEnemic = vidaEnemic - dany;
Console.WriteLine("Vida restant: " + vidaEnemic);

6. Exemple modularitzat 

public static int CalcularDany(int atac, int defensa) {
    int dany = atac - defensa;
    if (dany < 0) dany = 0;
    return dany;
}

public static int AplicarDany(int vidaActual, int dany) {
    return vidaActual - dany;
}

Ara el Main queda molt més net i professional:

int danyCalculat = CalcularDany(atacJugador, defensaEnemic);
vidaEnemic = AplicarDany(vidaEnemic, danyCalculat);

7. Avantatges de la modularitat

  • ✔ Evita repetir codi
  • ✔ Facilita el testing
  • ✔ Permet treball en equip
  • ✔ Millora la llegibilitat
  • ✔ Redueix errors lògics
  • ✔ Escalabilitat del joc
