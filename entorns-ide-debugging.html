🧠 Entorns IDE i Debugging

L'estació de combat del programador de videojocs

1. Què és un IDE?

Un IDE (Integrated Development Environment) és un entorn que integra totes les eines necessàries per programar en un sol lloc.

Normalment inclou:

  • >> Editor de codi
  • >> Compilador o intèrpret
  • >> Sistema de debugging
  • >> Gestor de projectes
  • >> Control de versions
  • >> Autocompletat (IntelliSense)

2. IDEs habituals en videojocs

🎮 Visual Studio

Ús principal: C# i Unity.

  • - IntelliSense molt potent
  • - Debugger complet
  • - Integració directa amb Unity
  • - Refactorització automàtica

🎮 Visual Studio Code

Ús principal: Editor lleuger amb extensions.

  • - Molt ràpid i lleuger
  • - Personalitzable amb extensions
  • - Ideal per equips menys potents

🎮 Unity

Unity no substitueix l'IDE, però inclou:

Consola d'errors
Inspector d'objectes
Mode Play (Test)

3. Funcionalitats clau d'un IDE

✍ Editor intel·ligent: Autocompletat, detecció d'errors en temps real i format automàtic.

🗂 Gestió de projectes: Organització en carpetes, referències i compilació automàtica.

🔄 Control de versions: Integració amb Git, seguiment de canvis i resolució de conflictes.

🧪 Debugger: La part més important a nivell formatiu per caçar bugs.

4. Què és el Debugging?

El debugging és el procés d’identificar i corregir errors (bugs) en un programa.

Exemples en videojocs:

  • - Mecàniques que no funcionen
  • - Personatge que travessa el terra (Noclip bug)
  • - Puntuacions incorrectes
  • - Errors de física o NullReferenceException

5. Tipus d'errors

❌ Errors de sintaxi

El codi no compila. L’IDE marca l’error abans d’executar.

int vida = "100"; // No es pot assignar string a int

⚠ Errors d'execució

El programa compila però falla mentre el jugues.

GameObject enemy = null;
enemy.transform.position = Vector3.zero; // NullReferenceException

🧠 Errors lògics

El programa funciona, però el resultat és incorrecte.

if (vida < 0) { GameOver(); } 
// Error: Hauria de ser vida <= 0 per incloure la mort

6. Eines de Debugging

🔴 Breakpoints

Un punt on el programa s'atura per analitzar l'estat.

  • - Veure valors de variables
  • - Executar línia a línia
  • - Analitzar el flux

👣 Execució pas a pas

Step Into: Entra dins del mètode.

Step Over: Executa sense entrar-hi.

Step Out: Surt del mètode actual.

👀 Watch / Variables locals

Permet observar canvis en temps real (vida, velocitat, IA) mentre el codi s'executa lentament.

SYSTEM_CLEANER: DEBUG_MAZE

BUGS_RESOLVED: 0

Fes servir les FLETXES del teclat per moure el depurador

Recull els errors (punts) i evita els BUGS (fantasmes) per estabilitzar el sistema.
