🧠 Git i GitHub – Control de Versions

El manual de supervivència del desenvolupador

1. Què és el Control de Versions?

Permet gestionar els canvis en fitxers i projectes al llarg del temps.

  • ✔ Guardar l’historial de canvis.
  • ✔ Tornar a versions anteriors (Checkpoints).
  • ✔ Treballar en equip sense conflictes.

👉 En videojocs és essencial: codi, escenes, scripts, assets... tot evoluciona!

2. Què és Git?

És un sistema de control de versions distribuït. Cada dev té una còpia completa de l'historial.

Les 4 àrees principals:

🗂 Working Directory

Carpeta local on editem fitxers.

📦 Staging Area

Preparació de canvis. git add

🧠 Local Repo (HEAD)

Historial confirmat. git commit

☁ Remote Repo

Repositori al núvol (GitHub).

3-4. Flux de Treball i Comandes

EDITAR → git add → git commit → git push

Inicialització

git init
git config --global user.name "Dani"
git config --global user.email "dani@cf.com"

Treball Diari

git status
git add .
git commit -m "Add new level logic"

5. Branches (Branques)

Permeten desenvolupar funcionalitats sense afectar la branca principal (main).

main |-- branca-inventari branca-IA 
git branch nova-funcio      // Crea
git checkout nova-funcio    // Canvia
git merge nova-funcio       // Fusiona

8. Bones pràctiques

✔ 1 commit = 1 canvi lògic

✔ Missatges clars i descriptius

✔ 1 branca per funcionalitat

✔ Fer pull abans de començar

✔ Utilitzar .gitignore per a assets binaris
